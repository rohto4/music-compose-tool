## Adopted for the comparison

- design-context gate: Audience / Use case / Toneをprototype前に確認する。
- working productをfirst viewportに置く。
- named macrostructureとthemeを明示し、連続案で構造を使い回さない。
- color/fontをtoken化し、途中でraw valueを増やさない。
- fabricated metric、fake testimonial、fake browser/phone chromeを使わない。
- interactive elementのdefault / hover / focus / active / disabled / loading / error / successを設計する。
- 320 / 375 / 414 / 768pxとreduced motionを確認する。
- handoff前にHallmarkのslop testとscope contractを読む。

## Not adopted automatically

- upstream codeやskill一式をこのPJの正本として複製しない。
- package、font、asset、analytics、外部serviceをHallmarkの例だけで追加しない。
- landing page向けmacrostructureを作曲workbenchへ機械的に当てはめない。
- Hallmark案をユーザー判断前にデザイン正本へしない。
